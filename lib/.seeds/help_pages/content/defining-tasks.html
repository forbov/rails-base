Defining Tasks

A task is defined by a client to define everything required to successfully complete the task including:

  • Classification of the task
  • How often it needs to be performed
  • Detailed description of the task
  • Equipment required to complete the task and where it is located
  • Time period applicable for the task
  • Specific days of the week for the task

A task can be forced to appear on the next activated shift

Tasks can only be added by Clients. To add a task click the 'Add Task' button from the Tasks tab on the Dashboard screen.

The following fields are mandatory:

  • Classification: The type of task to be performed
  • Frequency: How often the task is to be performed

The following fields are optional:

  • Start date: The date after which the task can appear on a shift. Ignored if blank.
  • End date: The date before which the task can appear on a shift. Ignored if blank.
  • Force: A flag which will force the task to appear on the next activated shift.
  • Specific Days A set of flags indicating the task is only relevant on particular days.
